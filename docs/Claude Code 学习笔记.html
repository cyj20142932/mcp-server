01

快速上手

💻 安装、登录、第一次对话

Claude Code 是一个跑在终端里的 AI 编程助手。你在终端里跟它对话，它帮你写代码、改代码、执行命令。

🔄 Claude Code 工作流程
📦 安装
npm install -g
→
📂 进入项目
cd my-project
→
🚀 启动
claude
→
💬 对话
自然语言交互
→
✨ 写代码
自动生成/修改
bash
# 1. 安装（需要先装 Node.js v18+）
npm install -g @anthropic-ai/claude-code

# 2. 进入你的项目目录，然后启动
cd my-project
claude

首次启动会提示登录，两种方式：

  • 订阅制（Claude Pro/Max 会员）— 适合日常使用
  • API Key（按量计费）— 适合重度用户，成本可控

选择后浏览器会弹出授权页面，点同意就行。之后可以用 /login 重新登录。

Claude Code 不绑定 Claude 模型，通过环境变量可以接入其他模型。学会 Claude Code 的操作逻辑后，Codex、Open Code 等同类工具也能触类旁通。

🎛️ 三种模式

Claude Code 有三种工作模式，按 Shift+Tab 循环切换。看终端底部的文字就知道当前是哪个模式：

🎛️ 三种工作模式
🛡️
默认模式
底部显示 ? for shortcuts
每次改文件前问你同不同意
新手推荐
⚡
自动模式
底部显示 Accept Edits On
自动改文件不问你
信任 Claude 时用
📐
规划模式
底部显示 Plan Mode
只讨论方案不动手
想清楚再干活
按 Shift+Tab 循环切换：默认 → 自动 → 规划 → 默认 …

默认模式下，Claude 要改文件时会弹出确认：

  • Yes — 同意这一次
  • Yes, all edits — 本次会话内都自动同意（等于切到自动模式）
  • No — 拒绝，可以告诉它怎么改
常用快捷键：Shift+Enter 换行（回车是提交），Ctrl+G 打开编辑器写长文本。

⌨️ 执行终端命令

在 Claude Code 里输入 ! 开头就能直接跑终端命令，不用退出：

bash
! ls                  # 看当前目录有什么文件
! open index.html     # 用浏览器打开文件（macOS）
! npm install         # 安装依赖
! git status          # 查看 Git 状态
不要在 Claude Code 里跑长时间运行的命令（如 npm run dev），会阻塞整个会话。开发服务器请在另一个终端窗口跑。
02

日常使用技能

📐 规划模式：大任务先想再做

想象你要把一个纯 HTML 页面重构成 React + TypeScript + Vite 项目。如果直接让 Claude Code 动手，它可能改到一半你才发现方向不对。

规划模式就是解决这个问题的 — 先让 Claude 出方案，你审核满意了再执行。

bash
# 进入规划模式：Shift+Tab 切到 Plan Mode，或输入 /plan

# 输入需求（Shift+Enter 换行）
> 将当前待办应用重构为 React + TypeScript + Vite
> 保留所有现有功能，UI 风格保持一致

Claude 会生成详细计划（目标、项目结构、步骤），然后给你三个选项：

  1. 执行计划 + 自动模式 — 直接干，不再问
  2. 执行计划 + 默认模式 — 边干边确认
  3. 继续修改计划 — 不满意，接着改

你可以反复修改计划直到满意。规划模式下 Claude 不会动任何文件，完全安全。

执行时建议选自动模式，避免频繁确认打断流程。

🔓 跳过权限检测

即使在自动模式下，执行终端命令（mkdir、npm install 等）时 Claude Code 仍然会问你。如果觉得烦，可以启动时加参数：

bash
claude --dangerously-skip-permissions

加了之后所有操作都不再确认。参数名里有 "dangerously" 是官方故意提醒你：这意味着 Claude Code 拥有和你一样的终端权限。

  • 个人项目、测试环境 → 放心用
  • 生产环境、团队项目 → 慎用
  • 折中方案：不加这个参数，但对常用命令选 "Yes, always for this command" 逐个授权

🔧 后台任务 + 版本回滚 + 会话恢复

这三个功能解决的是同一类问题：在 Claude Code 里工作时的"意外处理"。

📌 后台任务

如果 Claude Code 不小心启动了一个服务（比如 npm run dev），整个会话会被阻塞，你打字没反应。

解决办法：按 Ctrl+B 把服务丢到后台，然后继续正常对话。用 /tasks 查看后台任务，按 K 关掉。

⏪ 版本回滚

Claude Code 每次执行操作前会自动创建回滚点。后悔了？两种方式回滚：

bash
/rewind          # 输入命令
# 或连按两次 ESC

回滚界面会列出所有操作节点，选一个回退到那个时间点。回滚有三种粒度：

  • 代码 + 对话都回滚
  • 只回滚对话
  • 只回滚代码
Claude Code 只能回滚它自己写的文件。终端命令产生的文件（如 node_modules、package.json）回滚不了，需要手动删除。所以强烈建议配合 Git 使用，每完成一个功能就 git commit。

💾 会话恢复

退出 Claude Code 后对话不会丢。下次想接着聊：

bash
claude -c              # 直接恢复上一次对话（最常用）
claude --resume        # 从历史会话列表里选一个恢复

🖼️ 图片输入 + 上下文管理

🖼️ 传图片给 Claude Code

Claude Code 能看图。你可以传设计稿截图、报错截图、架构图，它都能分析。

两种传图方式：

  • 直接拖拽图片文件到终端窗口
  • 复制图片后按 Ctrl+V（macOS 也是 Ctrl+V，不是 Cmd+V！）
图片方式适合快速原型。如果要精确还原设计稿（字体、间距、颜色都对），用 MCP 接 Figma（下一部分会讲）。

📊 上下文管理

聊久了，Claude Code 的上下文会越来越大，响应变慢、Token 消耗增加。两个命令：

bash
/compact       # 压缩上下文：保留关键信息，丢弃冗余（有损压缩）
/clear         # 清空上下文：彻底重来，适合切换到不相关的新任务

压缩后按 Ctrl+O 可以看压缩结果。

📊 上下文管理示意
开始
对话中
聊很久
⚠️ 快满了
→
/compact
/clear

/compact 有损压缩保留关键信息 · /clear 彻底清空重来

上下文跟会话绑定，换个会话就没了。需要持久保存的信息（项目规范、技术栈等），写到 CLAUDE.md 里（下一部分讲）。
03

让 Claude Code 更懂你的项目

📝 CLAUDE.md：项目记忆文件

这是 Claude Code 最重要的配置之一。

打个比方：每次你启动 Claude Code，它就像一个新来的同事，对你的项目一无所知。CLAUDE.md 就是你给这个同事写的"项目交接文档"— Claude Code 每次启动都会自动读取它。

bash
/init          # 让 Claude Code 自动生成一份（可能是英文，可以让它翻译）
/memory        # 打开编辑（可选项目级或用户级）
🏗️ CLAUDE.md 作用范围
🌐 用户级 ~/.claude/CLAUDE.md
对你所有项目生效
📁 项目级 ./CLAUDE.md
只对当前项目生效 · 建议提交到 Git

适合写什么：项目架构、技术栈、编码规范、常用命令、注意事项。写什么 Claude Code 就遵循什么。

别写太多无关内容，每次启动都会加载，太长浪费 Token。

🔌 MCP：给 Claude Code 装"外挂"

MCP（Model Context Protocol）是让 Claude Code 连接外部工具的协议。你可以把它理解成"插件接口"— 装了 MCP Server，Claude Code 就多了新能力。

比如装了 Figma MCP Server，Claude Code 就能直接读取 Figma 设计稿的详细信息（间距、字体、颜色值），比看截图精确得多。

🔌 MCP 架构：Claude Code 连接外部世界
🤖 Claude Code
⟷
🔗 MCP Server
⟷
🎨 Figma
🗄️ 数据库
🌐 API

MCP 是标准协议，一个 Server 可以连接多种外部工具

📥 安装和使用 MCP

bash
# 1. 退出 Claude Code（Ctrl+C 两次）
# 2. 按照 MCP Server 的官方文档执行安装命令
# 3. 重新启动 Claude Code
claude

# 4. 查看和授权
/mcp           # 选择 MCP Server → Authenticate 完成授权

🎨 用 Figma MCP 还原设计稿

bash
# 在 Figma 中：右键 → Copy link to selection
# 在 Claude Code 中：
> 修改当前页面，使它与 Figma 设计稿保持一致
> https://www.figma.com/file/xxxxx/...

Claude Code 会自动调用 MCP 获取设计稿的详细样式信息，然后生成代码。比传截图精确得多。

除了 Figma，还有数据库、API、文件系统等各种 MCP Server 可用。配置保存在 .claude.json 文件中。
04

高级定制

⏱️ Hook：自动化工作流

Hook 让你在 Claude Code 执行操作前后自动运行 Shell 命令。

最典型的场景：Claude Code 每次创建或修改文件后，自动用 Prettier 格式化代码。

⏱️ Hook 触发时间线
PreToolUse
工具使用前
🔧 执行工具
Write / Edit
PostToolUse
工具使用后 ✨
ToolFailure
失败时触发

✨ PostToolUse 最常用 — 比如文件修改后自动格式化

bash
/hooks
# → 选择 PostToolUse（工具使用后触发）
# → 匹配工具名：Write|Edit
# → 输入命令：
echo "$CLAUDE_TOOL_RESULT" | jq -r '.filePath' | xargs prettier --write
# → 选择保存级别

保存级别：

  • 本地项目级（.claude/settings.local.json）— 不随 Git 分发，个人用
  • 项目级（.claude/settings.json）— 随 Git 分发，团队统一规范
  • 用户级（~/.claude/settings.json）— 对你所有项目生效
需要先安装 jq 和 prettier。Hook 命令出错不影响 Claude Code 正常运行。

🧩 Skill 和 SubAgent：扩展 Claude Code 的能力

这两个功能都是让 Claude Code 学会新技能，但工作方式不同。

🔀 Skill vs SubAgent

📜 Skill（技能）

共享上下文的 Prompt 模板

和主对话共享上下文 — Claude 能看到你之前聊了什么

适合：轻量级任务
如写日报、生成 changelog

🤖 SubAgent（子代理）

独立上下文的专属 Agent

有自己独立的上下文，干完活只把最终结果交回来

适合：重量级任务
如代码审核、大规模分析

📜 Skill 使用方式

bash
# 创建 Skill
mkdir -p ~/.claude/skills/daily-report
# 在里面创建 SKILL.md，写上名称、描述和格式要求

# 使用
/daily-report 写一份今天的总结    # 主动调用
# 或直接说"写一份日报"，Claude Code 会自动匹配

🤖 SubAgent 使用方式

bash
/agent
# → Create New Agent → 选级别 → 描述功能 → 选工具权限和模型

🤔 怎么选？

任务会产生大量中间结果（如审核几万行代码）？
SubAgent（不污染主上下文）
需要知道之前聊了什么？
Skill（如：根据今天的对话写日报）
SubAgent

简单记：Skill 共享上下文，SubAgent 独立上下文。

📦 Plugin：一键安装扩展包

Plugin 是 Skill、SubAgent、Hook、MCP 的打包组合，一键安装。

📦 Plugin = 能力打包
🧩 Plugin 扩展包
📜 Skill
🤖 SubAgent
⏱️ Hook
🔌 MCP
bash
/plugins
# → Discover 发现新插件 → 选择安装

比如 "Frontend Design" 插件，本质是一个包含 Anthropic 官方 UI 设计准则的 Skill，装了之后 Claude Code 生成的前端界面会好看很多。

你自己的配置写得好，也可以打包成 Plugin 分享给团队或社区。
